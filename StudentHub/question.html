Как реализовать JWT аутентификацию в Node.js с refresh токенами?

АК
Алексей Кузнецов 156
2 часа назад 234

Пытаюсь сделать полноценную auth систему с access и refresh токенами. Access живёт 15 минут, refresh — 7 дней. Не понимаю, как правильно организовать хранение refresh токенов на сервере и безопасно передавать их клиенту.

Вот пример кода, который я использую сейчас:

const jwt = require('jsonwebtoken');
function generateTokens(userId) {
  const accessToken = jwt.sign({ userId }, process.env.ACCESS_SECRET, { expiresIn: '15m' });
  const refreshToken = jwt.sign({ userId }, process.env.REFRESH_SECRET, { expiresIn: '7d' });
  return { accessToken, refreshToken };
}

Вопросы:

  • Где хранить refresh токены? В БД или Redis?
  • Как обновлять access токен через refresh?
  • Нужно ли удалять refresh токен при выходе?

Рекомендую использовать Redis для хранения refresh токенов. Вот правильная архитектура:

await redis.setex(`refresh:${userId}`, 7*24*60*60, refreshToken);

Преимущества: автоматическое удаление по TTL, быстрый доступ.

Я храню refresh токены в PostgreSQL. Для небольших проектов хватит. Вот структура таблицы:

CREATE TABLE refresh_tokens (
  user_id INT REFERENCES users(id),
  token TEXT UNIQUE,
  expires_at TIMESTAMP
);
